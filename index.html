Vue3起步——邂逅setup函数

在学习Vue3的时候，我们首先第一步要接触的肯定就是setup函数了。那么，setup函数是什么呢？

首先，我们要明确的是setup函数是所有组合式API的入口。那么问题又来了，什么是组合式API呢？
通过创建Vue组件，我们可以将界面中重复的部分连同其功能一起提取为可重用的代码段。仅此一项就可以使我们的应用在可维护性和灵活性方面走得相当远。然而，我们的经验已经证明，光靠这一点可能并不够，尤其是当你的应用变得非常大的时候——想想几百个组件。处理这样的大型应用时，共享和重用代码变得尤为重要。这是Vue3官方文档的解释，通过这段话我们知道，组合式API的特点就是将组件间公用的部分及代码抽离出来，然后变为一个可复用的代码段。这相对于Vue2的配置项API来说是一个大的改进，因为Vue2的配置项API会把所有的逻辑点全部写在一个文件里，如果逻辑少的话还算方便，但是当逻辑关注点多了以后，整个组件就会变得非常冗杂，可读性和可维护性就会大大降低。

那么组合式API的作用就出现了：将多个逻辑关注点的代码整合在一起，增强可读性。现在，我们知道了组合式API的作用，那它到底怎么整合呢？这就要交给setup函数来处理了。先来了解setup函数：

  • setup函数是所有组合式API的入口；
  • 它的生命周期在beforeCreate和Created之前；
  • 它自身有两个参数：第一个是props，使用这个参数之前必须在初始化的时候声明，（type,default,validator）,因为它自身是一个响应式数据，解构会让props失去响应式；
  • 第二个参数是context：这里包含定义的事件，和一些没有被声明的属性参数，attrs和slots在这里被调用；
  • 本身是一个函数，里面集合了组件的数据，方法等 ；
  • 有返回值，是一个对象，并且对象里的值可以直接在组件内使用。

git之团队内协作和跨团队协作

发表于 2021-08-24 更新于 2021-08-25 Changyan：

今天在学习git的时候，了解到两个很重要的概念：团队内协作和跨团队协作

团队内协作的机制

  • 本地库1和本地库2分别代表两个成员；
  • 成员1将写好的代码push到远程库，然后成员二想修改代码，这时候就通过git pull 或 git clone把它拉取到自己的本地库；
  • 修改好后，此时成员2想把修改后的代码提交到远程库，这时候通过git push 是不可以的；
  • 此时就需要成员1把成员2拉进自己本地库的管理团队中，这时成员2才有提交的权限；
  • 然后，成员2就可以顺利提交到远程库。

跨团队协作机制

  • 本地库1本地库2和本地库3分别代表三个成员，其中成员1和成员2互为团队成员；
  • 成员1将写好的代码推送到远程库，这时候团队外的一名成员想要修改代码；
  • 先在github中搜索到这个项目，然后再fork一份到自己的远程库；
  • 再通过git pull/git clone 拉取到自己的本地库
  • 将代码修改好后，push到自己的远程库
  • 这个时候要想推送给成员1的远程库，就要先给成员1发送pull Request
  • 成员1收到请求后，先查看代码是否符合自己的预期，如果符合预期，就同意申请
  • 同意后，再把成员3修改的代码merg到自己代码上
  • 之后成员1和成员2就可以看到修改后的代码

前记

其实我很久之前就想写自己的Blog了，但由于之前一直在自学前端，再加上学校的一些事情，写Blog就被渐渐抛到脑后。

那是什么原因让我又重新产生写Blog的想法了呢？其实很简单，就是我最近在投简历的时候发现很多HR都会问我有没有自己博客，作为一个非科班出身而且又是跨专业学习前端的我来说，写博客无疑是一件困难的事情（部分理科生的文采都不是很好）。

但同时我又发现，写博客对自己来说是一个很好的提升机会。它可以更好的巩固自己学到的东西，把自己在平时学习中遇到的问题分享出来，让大家一起想办法解决。

记得上初中的时候老师就说过，自己学会不是真正的会，要把它向别人讲出来才说明自己真的会了。所以以后我都会把自己在学习中的问题和想法分享给大家。
希望大家可以一起努力，一起进步!
